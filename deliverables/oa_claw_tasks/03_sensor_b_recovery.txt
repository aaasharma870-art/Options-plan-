CONTEXT
=======================================================================

Sensor B detects the transition from RED yesterday → not-RED today. When
that happens, it sets the REGIME_RECOVERY tag. Bot 6 (ic-regime-recovery)
uses this tag as its entry trigger.

OA has no "yesterday" lookup natively. The workaround: Sensor A leaves
a shadow tag REGIME_RED_YESTERDAY each day it tags RED. Sensor B (running
1 minute later at 9:46 AM) inspects both tags:
- If REGIME_RED_YESTERDAY is present AND REGIME_RED is NOT → recovery!
- If REGIME_RED_YESTERDAY is present AND REGIME_RED is still present →
  still RED, keep the shadow tag for tomorrow.

=======================================================================
STEP 1: Create Shared Event Automation
=======================================================================

1. Inside SYSTEM - Regime Sensor Host bot, click "+ New Automation".
2. Type: EVENT
3. Name: Sensor B - Recovery Detector
4. Mark as Shared Automation
5. Trigger: Recurring, 9:46 AM ET, Mon-Fri
   NOTE: This is one minute AFTER Sensor A. The delay is intentional
   to guarantee Sensor A's tags are fresh when Sensor B reads them.
6. SCREENSHOT: trigger configured.

=======================================================================
STEP 2: Build the Decision Flow
=======================================================================

In the flow editor, build from top to bottom:

Action 1: Untag Bot
  Tag name: REGIME_RECOVERY
  (Clear any stale recovery tag from previous days.)

Decision 1: Bot has tag
  Tag name: REGIME_RED_YESTERDAY

SCREENSHOT.

On YES branch of Decision 1:
  Decision 2 (nested): Bot does NOT have tag
    Tag name: REGIME_RED

SCREENSHOT.

On YES branch of Decision 2 (i.e., was RED yesterday, not today):
  Action: Tag Bot → REGIME_RECOVERY
  Action: Untag Bot → REGIME_RED_YESTERDAY (cleanup, transition done)

On NO branch of Decision 2 (i.e., was RED yesterday, still RED today):
  Action: Untag Bot → REGIME_RED_YESTERDAY
  Action: Tag Bot → REGIME_RED_YESTERDAY
  (This removes and re-adds the shadow tag — effectively a no-op but
  ensures it persists to tomorrow for the next check.)

On NO branch of Decision 1 (not RED yesterday):
  No action needed — REGIME_RECOVERY stays cleared.

SCREENSHOT: complete flow.
