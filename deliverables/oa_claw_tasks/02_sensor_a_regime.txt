=======================================================================
CONTEXT
=======================================================================

This is the single most important automation in the entire system.
It runs every morning at 9:45 AM ET and tags every attached bot with
one of four regime labels (REGIME_GREEN / ORANGE / ORANGE_PLUS / RED)
based on current VIX metrics. Every trading bot reads this tag to
decide whether to enter positions today.

This task has the most decision nodes of any in the series. Take your
time. Screenshot at every decision creation. If unsure about a node,
STOP rather than guess.

=======================================================================
SAFETY RULES
=======================================================================

1. PAPER mode
2. You are building INSIDE the SYSTEM - Regime Sensor Host bot
3. Mark the automation as "Shared" — this is CRITICAL. Non-shared
   automations cannot be attached to other bots.
4. Screenshot after each decision node creation
5. If OA's decision node types don't match the ones described (e.g.,
   no "Symbol Price Change" decision, or OR logic works differently),
   STOP and describe what's available

STEP 3: Build the Decision Flow — Part 1: Reset Tags
=======================================================================

In the flow editor, add these 4 actions in sequence at the top:

Action 1: Untag Bot
  Tag name: REGIME_GREEN

Action 2: Untag Bot
  Tag name: REGIME_ORANGE

Action 3: Untag Bot
  Tag name: REGIME_ORANGE_PLUS

Action 4: Untag Bot
  Tag name: REGIME_RED

SCREENSHOT: flow showing all 4 untag actions at the top.

RATIONALE: We reset all possible regime tags at the start of each run
so only the tag for today's regime remains after the decision tree fires.

=======================================================================
STEP 4: Decision 1 — RED Branch (VIX > 30)
=======================================================================

After the 4 untags, click "+ Add Decision".

Decision type: Symbol Indicator
  Symbol: VIX
  Indicator: Last price
  Comparison: is greater than
  Value: 30

SCREENSHOT: decision configured.

On the YES branch of this decision, add 3 actions:

Action 1: Tag Bot
  Tag name: REGIME_RED

Action 2: Tag Bot
  Tag name: REGIME_RED_YESTERDAY
  (NOTE: This is a shadow tag used by Sensor B tomorrow.)

Action 3: Tag Bot
  Tag name: RED_D1
  (NOTE: Day-streak counter — optional but recommended.)

SCREENSHOT: YES branch completed with all 3 tags.

PAUSE POINT 2: Confirm the YES branch has exactly 3 tag actions.

=======================================================================
STEP 5: Decision 2 — ORANGE_PLUS Branch (compound OR)
=======================================================================

On the NO branch of Decision 1, click "+ Add Decision".

This decision uses OR logic — three conditions, any of which triggers
ORANGE_PLUS. Build as a COMPOUND decision if OA supports it, or as
nested decisions if not.

CONDITION A (VIX > 25):
  Decision type: Symbol Indicator
  Symbol: VIX
  Indicator: Last price
  Comparison: is greater than
  Value: 25

Click "+ Add OR Condition" (or equivalent in OA UI).

CONDITION B (VIX > VIX3M × 1.05):
  Decision type: Symbol Comparison
  First symbol: VIX
  Comparison: is greater than
  Second symbol: VIX3M
  Multiplier on second symbol: × 1.05

Click "+ Add OR Condition".

CONDITION C (VIX spiked):
  Decision type: Symbol Price Change
  Symbol: VIX
  Timeframe: 5 days ago
  Comparison: has increased by at least
  Measure: 2 standard deviations (30-day std dev)

SCREENSHOT: compound decision with all 3 OR conditions.

On the YES branch of this compound decision:

Action 1: Tag Bot
  Tag name: REGIME_ORANGE_PLUS

SCREENSHOT.
